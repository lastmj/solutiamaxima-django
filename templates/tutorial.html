Solutia Maxima - Tutorial

Thanks for testing with us! We want to get you up and creating problems as soon as possible. If you want a little background on the vision for Solutia Maxima, keep reading. Otherwise, go straight down to Getting Started to start creating problems right away.

Why Solutia Maxima?

We envision a website where anyone studying anything can come to practice their skills. This practice will include solving problems and being able to see the answers to those problems, with ease. We want everyone to be able to find exactly what they need to hone their skills. Once they find that perfect problem, they’ll be able to practice it over and over and over, until finally they have mastered the concept. But, what if the problem does not exist? I’m glad you asked. All are free to come and create problems at will. Together we can cover an amazing breadth of knowledge, and put it into a form that others can use to their benefit. All are free to learn, all are free to create. That is why.

JavaScript Basics

Variables

Nearly everything you do while coding your problems will take place within variables. Variables are your friends. They allow you to store words and numbers and other things for later use. There are two basic steps you need to know to use a variable. First you give it a name, then you give it a value:

monkeys = 50

Now our variable monkeys contains the value 50. Whenever we need that value later on, we can simply use monkeys instead of 50. It gets really useful when you store more than just one number in a variable:

bananas = 20
monkeys = 50 + bananas

Now monkeys contains the sum of 50 and whatever bananas contains inside of it, which is the value 20. Variables can also hold words. We call them strings:

firstName = "Johnny"

firstName now contains the string with the value of “Johnny”. You can add two string variables together to get a different string:

firstName = "Johnny"
lastName = "Williams"
fullName = firstName + lastName

What do you think fullName contains now? Exactly right, “JohnnyWilliams”. What if we want to make it look nice by adding a space?

firstName = "Johnny"
lastName = "Williams"
fullName = firstName + " " + lastName

Now fullName’s value is “Johnny Williams”. I think you get the idea. Now, how would you combine strings and numbers (variables holding number values instead of string values)?

firstHalf = "I have"
numApples = 50
secondHalf = " apples"
completeSentence = firstHalf + numApples.toString() + secondHalf

Now completeSentence’s value is "I have 50 apples". Whenever you want to add a number to your string, you need to use the .toString() method to make it compatible for use with strings.

And finally, remember that when coding for Solutia Maxima, you always want your variables to be global; basically, don’t put anything in front of your variable names when you are using them. If you don’t know what we’re talking about, don’t worry about it, you’ll be just fine.

If...else Statements

One of the beauties of programming is that you can teach your programs to make their own decisions. You do that using conditional statements, probably more commonly known as if...else statements. Let’s learn:

test = 5

if(test == 5)
{
    test = 50
}

What will the result of test be after this code runs? That’s right! The result will be 50. Remember that inside of an if statement you need == to test for equality:

test = "Jimmy"

if(test == "Steve")
{
    test = "equal"
}
else
{
    test = "not equal"
}

What does test hold now? Incredible! The value of test is “not equal”. So what happened? Our if statement returned false, and so our code skipped down to the accompanying else statement and ran the code in there, thus setting test equal to “not equal”. Let’s cover one more variation:

myNumber = 600

if(myNumber != 600)
{
    myNumber = 10
}
else if (myNumber == 300 && myNumber == 2)
{
    myNumber = 30
}
else
{
    myNumber = 5
}

The final value of myNumber is...5. Why? Because. Let’s explain. The first if statement failed because myNumber was equal to 600. It did not not equal 600. So, we went to the else if statement, which only runs if the statement above it returns false. The else if also failed, because myNumber did not equal 300 and certainly did not equal 2. The && is the conditional AND operator. Both conditions of the else if needed to be true for the else if to return true, because of the && that we placed there. So, everything failed, and the last else statement was executed, causing myNumber to be set to 5. We have done it. Good job my friend.

Here’s a list of conditional operators you can use in your if...else statements:

== // equals: if(5 == 5) evaluates to true
!= // does not equal: if(50 != 30) evaluates to true
! // not: test = true
  //      if(!test) evaluates to false
&& // AND: if(11 == 11 && 34 == 34) evaluates to true
|| // OR: if(“monkey” == “orange” || “fun” == “fun”) evaluates to true
> // greater than:
>= // greater than or equal to:
< // less than
<= less than or equal to
